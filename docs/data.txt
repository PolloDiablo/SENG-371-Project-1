REDDIT POST DATA:
Can be parsed for the following information:
- gameName (which game this post is associated with)
- url (max: ~400 char) - guarenteed to be unique, this will be the table key
- date (date and time)
- title (max: 300 char)
- body (max: 10000 char)
- numberOfComments (unsigned integer)
- popularity (signed integer)

Which posts should be analzed for data?
- Analyzing all posts would be expensive (in time) and would yield a lot of data that will be thrown out anyway
- Instead, only analyze posts which contain keywords: "bug", "issue", "balance", etc.

PATCH NOTES DATA:
Can be parsed for the following information:
- gameName (which game this patch is associated with)
- url (max: ~400 char) - guarenteed to be unique, this will be the table key
- date (date and time)
- patch number or patch name (char/number mix), if applicable
- body (long text)
